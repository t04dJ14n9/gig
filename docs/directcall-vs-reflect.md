# Why DirectCall Wrappers Are ~10x Faster Than reflect.Call

## Executive Summary

When Gig calls an external Go function (e.g., `strconv.Atoi`, `strings.Contains`), it has two dispatch paths:

| Path | Mechanism | Overhead per call |
|---|---|---|
| **DirectCall** (fast) | Typed Go function wrapper, no reflection | ~50–100 ns |
| **reflect.Call** (slow) | `reflect.Value.Call([]reflect.Value)` | ~500–1000 ns |

The ~10x gap comes from **six compounding sources of overhead** in the reflection path. This document dissects each one with **actual Go standard library source code** from Go 1.23.

---

## 1. Concrete Example: `strconv.Atoi`

### DirectCall path (what actually runs)

```go
// Generated by gentool — stdlib/packages/strconv.go:222
func direct_strconv_Atoi(args []value.Value) value.Value {
    a0 := args[0].String()               // (1) Extract: read obj field → type assert to string
    r0, r1 := strconv.Atoi(a0)           // (2) Call: direct static call, fully inlinable
    return value.MakeValueSlice(          // (3) Pack result
        []value.Value{
            value.MakeInt(int64(r0)),     //     store int in num field (no alloc)
            value.FromInterface(r1),      //     wrap error in interface
        })
}
```

### reflect.Call path (what would run without DirectCall)

```go
// vm/call.go:388 — callExternalReflect
func (v *vm) callExternalReflect(entry *extCallCacheEntry, args []value.Value) error {
    in := buildReflectArgs(args, entry.fnType) // (1) Convert each arg → reflect.Value
    out := entry.fn.Call(in)                    // (2) reflect.Call — the expensive part
    v.push(value.MakeFromReflect(out[0]))       // (3) Convert each result back
    return nil
}
```

Let's trace **every CPU cycle** through both paths.

## 2. Prerequisite: How `reflect.Value` Works Internally

Before diving into the overhead, you need to understand the `reflect.Value` struct — the fundamental unit of Go reflection.

### 2.1 The `reflect.Value` Struct (24 bytes)

```go
// Source: reflect/value.go (Go 1.23)

type Value struct {
    // typ_ holds the type of the value represented by a Value.
    // Access using the typ method to avoid escape of v.
    typ_ *abi.Type

    // Pointer-valued data or, if flagIndir is set, pointer to data.
    // Valid when either flagIndir is set or typ.pointers() is true.
    ptr unsafe.Pointer

    // flag holds metadata about the value.
    //
    // The lowest five bits give the Kind of the value, mirroring typ.Kind().
    //
    // The next set of bits are flag bits:
    //  - flagStickyRO: obtained via unexported not embedded field, so read-only
    //  - flagEmbedRO: obtained via unexported embedded field, so read-only
    //  - flagIndir: val holds a pointer to the data
    //  - flagAddr: v.CanAddr is true (implies flagIndir and ptr is non-nil)
    //  - flagMethod: v is a method value.
    // If ifaceIndir(typ), code can assume that flagIndir is set.
    //
    // The remaining 22+ bits give a method number for method values.
    // If flag.kind() != Func, code can assume that flagMethod is unset.
    flag
}

type flag uintptr

const (
    flagKindWidth        = 5 // there are 27 kinds
    flagKindMask    flag = 1<<flagKindWidth - 1  // 0x1F
    flagStickyRO    flag = 1 << 5                // 0x20
    flagEmbedRO     flag = 1 << 6                // 0x40
    flagIndir       flag = 1 << 7                // 0x80
    flagAddr        flag = 1 << 8                // 0x100
    flagMethod      flag = 1 << 9                // 0x200
    flagMethodShift      = 10
    flagRO          flag = flagStickyRO | flagEmbedRO
)
```

**Key insight**: Every `reflect.Value` carries a **full type descriptor pointer** (`*abi.Type`) and metadata flags. Every operation on a `reflect.Value` must first check these flags. Compare this to Gig's `value.Value` which is a 32-byte tagged union that stores primitives directly in a `int64` field — no pointer chasing needed.

### 2.2 The `abi.Type` Struct (the runtime type descriptor)

Every Go type has a corresponding `abi.Type` in the binary. This is what `reflect.Value.typ_` points to:

```go
// Source: internal/abi/type.go (Go 1.23, simplified)

type Type struct {
    Size_       uintptr  // size of the type in bytes
    PtrBytes    uintptr  // number of (prefix) bytes that contain pointers
    Hash        uint32   // hash of the type; avoids computation in hash tables
    TFlag       TFlag    // extra type information flags
    Align_      uint8    // alignment of variable with this type
    FieldAlign_ uint8    // alignment of struct field with this type
    Kind_       Kind     // enumeration for C (reflect.Kind == abi.Kind)
    // function for comparing objects of this type (ptr to object A, ptr to object B) -> ==?
    Equal func(unsafe.Pointer, unsafe.Pointer) bool
    // GCData stores the GC type data for the garbage collector.
    GCData    *byte
    Str       NameOff   // string form (offset into string table)
    PtrToThis TypeOff   // type for pointer to this type (offset into type table)
}
```

Every reflection operation must chase from `reflect.Value.typ_` → `abi.Type` → read fields like `Kind_`, `Size_`, etc. This is pointer indirection that a direct call never needs.

## 3. The Six Sources of Overhead

### 3.1 — Argument Boxing: `value.Value` → `reflect.Value` (per argument)

**DirectCall:**
```go
a0 := args[0].String()  // Field read: v.obj.(string) → ~2 ns
```
`value.Value` is a 32-byte tagged union. For `KindString`, the string is stored directly in the `obj` field as a `string` (not boxed in an interface). Extraction is a single interface type assertion — typically 1-2 machine instructions.

**reflect.Call:**
```go
// model/value/accessor.go:226 — ToReflectValue for KindString
rv := reflect.ValueOf(v.obj.(string))   // Step A: extract the string
if rv.Type() != typ {
    rv = rv.Convert(typ)                 // Step B: type check + possible conversion
}
return rv
```

Each `reflect.ValueOf(x)` internally does:

```go
// Source: reflect/value.go (Go 1.23)

// ValueOf returns a new Value initialized to the concrete value
// stored in the interface i. ValueOf(nil) returns the zero Value.
func ValueOf(i any) Value {
    if i == nil {
        return Value{}
    }
    return unpackEface(i)           // ← the real work
}

// unpackEface converts the empty interface i to a Value.
func unpackEface(i any) Value {
    e := (*abi.EmptyInterface)(unsafe.Pointer(&i))
    // NOTE: don't read e.word until we know whether it is really a pointer or not.
    t := e.Type
    if t == nil {
        return Value{}
    }
    f := flag(t.Kind())             // ← extract Kind from type descriptor
    if t.IfaceIndir() {
        f |= flagIndir              // ← set "indirect" flag
    }
    return Value{t, e.Data, f}      // ← construct the 24-byte Value
}
```

**What happens step by step:**

1. **Interface boxing**: The caller writes `reflect.ValueOf(myString)`. The Go compiler must first box `myString` into an `any` (empty interface). An `any` is:
   ```go
   // Source: internal/abi/iface.go
   type EmptyInterface struct {
       Type *Type           // pointer to the type descriptor
       Data unsafe.Pointer  // pointer to the actual data
   }
   ```
   For a `string`, the compiler copies the 16-byte string header (data pointer + length) to heap or stack, then stores a pointer to it in `Data`. This is the **interface boxing cost** — it turns a value-type into a pointer-indirected type.

2. **`unpackEface`**: Reinterprets the `any` as `EmptyInterface`, extracts the `*abi.Type` pointer, reads `t.Kind()` (which requires loading `Type.Kind_` through the type pointer), computes the `flag` bitmask, and constructs a `reflect.Value{typ, ptr, flag}`.

3. **Type comparison**: Back in Gig's `ToReflectValue`, we check `rv.Type() != typ`:
   ```go
   // Source: reflect/value.go
   func (v Value) Type() Type {
       if v.flag != 0 && v.flag&flagMethod == 0 {
           return toRType(v.typ())   // fast path: just wrap the *abi.Type
       }
       // slow path: method value, must resolve method type...
   }
   ```
   Then `!=` compares two `Type` interfaces — which in the fast path is just a pointer comparison of `*abi.Type`. On mismatch, `rv.Convert(typ)` is called, which invokes a `convertOp` lookup.

**Cost**: ~15–30 ns per argument due to interface boxing, pointer chasing through `abi.Type`, and flag computation. For N arguments, this is **O(N) × 15 ns**.

### 3.2 — `[]reflect.Value` Allocation (one per call)

**DirectCall:**
```go
// No intermediate slice needed — args are already []value.Value
// The wrapper reads them directly by index.
```

**reflect.Call:**
```go
// vm/call.go:336 — buildReflectArgs
in := make([]reflect.Value, numArgs)  // HEAP ALLOCATION: 24 bytes × numArgs
for i, arg := range args {
    in[i] = toReflectArg(arg, fnType.In(i))
}
```

This `make([]reflect.Value, N)` call:
1. **Heap allocation**: `reflect.Value` is 24 bytes (typ pointer + ptr + flag). For 3 arguments → 72 bytes allocated on the heap.
2. **GC pressure**: Every call creates a new slice that becomes garbage immediately after `Call()` returns. The Go garbage collector must eventually scan and collect it.
3. **Cache pollution**: Allocations touch memory the L1 cache didn't have, evicting useful data.

In microbenchmarks, `make([]reflect.Value, N)` alone costs ~30–50 ns per call due to `runtime.mallocgc`.

### 3.3 — Runtime Type Checking Inside `reflect.Call` (per argument)

This is the **biggest single overhead**. When you call `fn.Call(args)`, the Go runtime does not trust you. It must verify every argument.

Here is the **actual Go source code** for `Value.Call` and its internal `call` method:

```go
// Source: reflect/value.go (Go 1.23)

// Call calls the function v with the input arguments in.
func (v Value) Call(in []Value) []Value {
    v.mustBe(Func)
    v.mustBeExported()
    return v.call("Call", in)
}

// mustBe panics if f's kind is not the expected kind.
func (f flag) mustBe(expected Kind) {
    if Kind(f&flagKindMask) != expected {
        panic(&ValueError{valueMethodName(), f.kind()})
    }
}

// mustBeExported panics if the value was obtained using an unexported field.
func (f flag) mustBeExported() {
    if f == 0 || f&flagRO != 0 {
        f.mustBeExportedSlow()
    }
}
```

Already **two runtime checks** before any real work. Now the `call` method:

```go
// Source: reflect/value.go (Go 1.23) — the full internal call method

func (v Value) call(op string, in []Value) []Value {
    // Get function pointer, type.
    t := (*funcType)(unsafe.Pointer(v.typ()))
    var (
        fn       unsafe.Pointer
        rcvr     Value
        rcvrtype *abi.Type
    )
    if v.flag&flagMethod != 0 {
        rcvr = v
        rcvrtype, t, fn = methodReceiver(op, v, int(v.flag)>>flagMethodShift)
    } else if v.flag&flagIndir != 0 {
        fn = *(*unsafe.Pointer)(v.ptr)
    } else {
        fn = v.ptr
    }

    if fn == nil {
        panic("reflect.Value.Call: call of nil function")
    }

    // ── PHASE 1: Validate argument count ──
    isSlice := op == "CallSlice"
    n := t.NumIn()
    isVariadic := t.IsVariadic()
    if isSlice {
        if !isVariadic {
            panic("reflect: CallSlice of non-variadic function")
        }
        if len(in) < n {
            panic("reflect: CallSlice with too few input arguments")
        }
        if len(in) > n {
            panic("reflect: CallSlice with too many input arguments")
        }
    } else {
        if isVariadic {
            n--
        }
        if len(in) < n {
            panic("reflect: Call with too few input arguments")
        }
        if !isVariadic && len(in) > n {
            panic("reflect: Call with too many input arguments")
        }
    }

    // ── PHASE 2: Validate each argument is not zero ──
    for _, x := range in {
        if x.Kind() == Invalid {
            panic("reflect: " + op + " using zero Value argument")
        }
    }

    // ── PHASE 3: Type-check each argument ──
    for i := 0; i < n; i++ {
        if xt, targ := in[i].Type(), t.In(i); !xt.AssignableTo(toRType(targ)) {
            panic("reflect: " + op + " using " + xt.String() + " as type " + stringFor(targ))
        }
    }

    // ── PHASE 4: Handle variadic arguments ──
    if !isSlice && isVariadic {
        m := len(in) - n
        slice := MakeSlice(toRType(t.In(n)), m, m)
        elem := toRType(t.In(n)).Elem()
        for i := 0; i < m; i++ {
            x := in[n+i]
            if xt := x.Type(); !xt.AssignableTo(elem) {
                panic("reflect: cannot use " + xt.String() + " as type " + elem.String() + " in " + op)
            }
            slice.Index(i).Set(x)
        }
        origIn := in
        in = make([]Value, n+1)    // ← ANOTHER allocation for variadic!
        copy(in[:n], origIn)
        in[n] = slice
    }

    // ... (ABI frame construction follows — see Section 3.4)
}
```

**Phase 3** is the critical one. For each argument, `AssignableTo` does:

```go
// Source: reflect/type.go (Go 1.23)

func (t *rtype) AssignableTo(u Type) bool {
    if u == nil {
        panic("reflect: nil type passed to Type.AssignableTo")
    }
    uu := u.common()
    return directlyAssignable(uu, t.common()) || implements(uu, t.common())
}
```

Two sub-checks are chained with short-circuit `||`:

#### `directlyAssignable` — Structure comparison

```go
// Source: reflect/type.go (Go 1.23)

// directlyAssignable reports whether a value x of type V can be
// directly assigned (using memmove) to a value of type T.
// https://go.dev/ref/spec#Assignability
// Ignoring the interface rules (implemented elsewhere)
// and the determine whether T and V have identical underlying types.
func directlyAssignable(T, V *abi.Type) bool {
    // x's type V is identical to T?
    if T == V {
        return true          // ← FAST PATH: pointer comparison
    }

    // Otherwise at least one of T and V must not be defined
    // and they must have the same kind.
    if T.HasName() && V.HasName() || T.Kind() != V.Kind() {
        return false
    }

    // x's type T and V must have identical underlying types.
    return haveIdenticalUnderlyingType(T, V, true)
}
```

When types don't match by pointer (common when crossing package boundaries), it falls through to `haveIdenticalUnderlyingType`:

```go
// Source: reflect/type.go (Go 1.23, simplified)

func haveIdenticalUnderlyingType(T, V *abi.Type, cmpTags bool) bool {
    if T == V {
        return true
    }

    kind := Kind(T.Kind())
    if kind != Kind(V.Kind()) {
        return false
    }

    // Non-composite types of equal kind are identical
    // (e.g., int vs int, string vs string).
    switch kind {
    case Bool, Int, Int8, Int16, Int32, Int64,
        Uint, Uint8, Uint16, Uint32, Uint64, Uintptr,
        Float32, Float64, Complex64, Complex128, String, UnsafePointer:
        return true

    case Array:
        return T.Len() == V.Len() &&
            haveIdenticalType(T.Elem(), V.Elem(), cmpTags)

    case Chan:
        return V.ChanDir() == T.ChanDir() &&
            haveIdenticalType(T.Elem(), V.Elem(), cmpTags)

    case Func:
        tt := (*funcType)(unsafe.Pointer(T))
        vt := (*funcType)(unsafe.Pointer(V))
        if tt.NumIn() != vt.NumIn() || tt.NumOut() != vt.NumOut() ||
            tt.IsVariadic() != vt.IsVariadic() {
            return false
        }
        for i := 0; i < tt.NumIn(); i++ {
            if !haveIdenticalType(tt.In(i), vt.In(i), cmpTags) {
                return false
            }
        }
        for i := 0; i < tt.NumOut(); i++ {
            if !haveIdenticalType(tt.Out(i), vt.Out(i), cmpTags) {
                return false
            }
        }
        return true

    case Map:
        return haveIdenticalType(T.Key(), V.Key(), cmpTags) &&
            haveIdenticalType(T.Elem(), V.Elem(), cmpTags)

    case Pointer, Slice:
        return haveIdenticalType(T.Elem(), V.Elem(), cmpTags)

    case Struct:
        tt := (*structType)(unsafe.Pointer(T))
        vt := (*structType)(unsafe.Pointer(V))
        if tt.NumField() != vt.NumField() {
            return false
        }
        for i := range tt.Fields {
            tf := &tt.Fields[i]
            vf := &vt.Fields[i]
            if tf.Name.Name() != vf.Name.Name() {
                return false
            }
            if !haveIdenticalType(tf.Typ, vf.Typ, cmpTags) {
                return false
            }
            if cmpTags && tf.Name.Tag() != vf.Name.Tag() {
                return false
            }
            if tf.Offset != vf.Offset {
                return false
            }
            if tf.Name.IsEmbedded() != vf.Name.IsEmbedded() {
                return false
            }
        }
        return true
    }
    return false
}
```

If `directlyAssignable` fails, `implements` is called — which checks interface satisfaction:

#### `implements` — Interface method-set scanning

```go
// Source: reflect/type.go (Go 1.23, simplified)

// implements reports whether the type V implements the interface type T.
func implements(T, V *abi.Type) bool {
    if T.Kind() != abi.Interface {
        return false
    }
    t := (*interfaceType)(unsafe.Pointer(T))
    if len(t.Methods) == 0 {
        return true  // every type implements the empty interface
    }

    // The same algorithm applies in both cases, but the method tables
    // for an interface type and a concrete type are different.
    // In both cases the second table is keyed like V.Methods.
    //
    // For a concrete type V, the methods are sorted by hash code.
    // For an interface type V, the methods are also sorted by hash code.
    //
    // Both methods tables are sorted by name, so we can merge-scan them.
    // This is O(len(T.methods) + len(V.methods)).
    
    var (
        vmethods []abi.Method
    )
    if V.Kind() == abi.Interface {
        vt := (*interfaceType)(unsafe.Pointer(V))
        vmethods = vt.Methods
    } else {
        // concrete type: walk the exported method list
        // ...
    }

    i := 0
    for j := 0; j < len(vmethods); j++ {
        tm := &t.Methods[i]
        vm := &vmethods[j]
        if vm.Name == tm.Name && vm.Type == tm.Type {
            i++
            if i >= len(t.Methods) {
                return true
            }
        }
    }
    return false
}
```

**This is a merge-scan over two sorted method lists** — O(M+N) where M and N are the method counts of the interface and the concrete type. For simple types like `string` or `int`, `directlyAssignable` returns `true` immediately (pointer equality). But for named types from different packages, it can fall through to `haveIdenticalUnderlyingType` which recursively walks the type structure.

**DirectCall does NONE of this**. The wrapper is compiled with static types — the Go compiler has already verified type safety at build time. Calling `args[0].String()` is a direct field read, not a runtime type check.

**Cost**: ~20–40 ns per argument for the full type-check loop.

### 3.4 — ABI Frame Construction and `reflectcall` (the call itself)

This is where `reflect.Call` fundamentally differs from a normal Go function call. Here is the **actual source code** from Go 1.23:

**DirectCall:**
```go
r0, r1 := strconv.Atoi(a0)
```
This compiles to a direct `CALL` instruction to `strconv.Atoi`'s address. Arguments are passed in registers (Go 1.17+ register-based ABI). The CPU's branch predictor knows this call target. It's **exactly** as fast as any normal Go function call — typically 2–5 ns for the call/return mechanism itself.

**reflect.Call — the actual source code (continued from Phase 4 above):**

```go
// Source: reflect/value.go (Go 1.23) — continued from v.call()

    if nin != t.NumIn() {
        panic("reflect.Value.Call: wrong argument count")
```

#### `funcLayout` — ABI layout computation (cached via sync.Map)

```go
// Source: reflect/type.go (Go 1.23)

// funcLayout computes a struct type representing the layout of the
// stack-assigned function arguments and return values for the function
// type t.
func funcLayout(t *funcType, rcvr *abi.Type) (
    frametype *abi.Type, framePool *sync.Pool, abid abiDesc,
) {
    k := layoutKey{t, rcvr}
    if lti, ok := layoutCache.Load(k); ok {        // ← sync.Map lookup!
        lt := lti.(layoutType)
        return lt.t, lt.framePool, lt.abid
    }

    // Compute the ABI details: which args go in registers, which on stack
    abid = newAbiDesc(t, rcvr)

    // Build a dummy abi.Type representing the frame
    // (needed for GC to scan the frame correctly)
    x := &abi.Type{
        Align_:   goarch.PtrSize,
        // arm64 has 16-byte aligned stack frames
        Size_:    align(abid.retOffset+abid.ret.stackBytes, goarch.PtrSize),
        PtrBytes: uintptr(abid.stackPtrs.n) * goarch.PtrSize,
    }
    // ... builds GC bitmap ...

    framePool = &sync.Pool{New: func() any {
        return unsafe_New(x)
    }}

    layoutCache.Store(k, layoutType{
        t:         x,
        framePool: framePool,
        abid:      abid,
    })
    return x, framePool, abid
}

var layoutCache sync.Map // map[layoutKey]layoutType
```

Even cached, `sync.Map.Load` costs 5-10 ns (atomic load + interface type assertion).

#### Frame allocation

```go
// Source: reflect/value.go (Go 1.23) — continued

    // ── PHASE 6: Allocate stack frame ──
    var stackArgs unsafe.Pointer
    if frametype.Size() != 0 {
        if nout == 0 {
            stackArgs = framePool.Get().(unsafe.Pointer)  // ← pool reuse (no return values)
        } else {
            // Can't use pool if the function has return values.
            // We will leak pointer to args in ret, so its lifetime is not scoped.
            stackArgs = unsafe_New(frametype)              // ← HEAP ALLOCATION!
        }
    }
    frameSize := frametype.Size()
```

**Critical**: For functions with return values (the common case!), `framePool` **cannot be used** because the returned `reflect.Value` structs may point into the frame. So `unsafe_New(frametype)` allocates a new chunk of memory every call. This is the **second heap allocation**.

#### Argument marshaling into the ABI frame

```go
// Source: reflect/value.go (Go 1.23) — continued

    // ── PHASE 7: Copy inputs into args ──

    // Handle receiver (for method calls).
    inStart := 0
    if rcvrtype != nil {
        // Guaranteed to only be one word in size,
        // so it will only take up exactly 1 abiStep.
        switch st := abid.call.steps[0]; st.kind {
        case abiStepStack:
            storeRcvr(rcvr, stackArgs)
        case abiStepPointer:
            storeRcvr(rcvr, unsafe.Pointer(&regArgs.Ptrs[st.ireg]))
            fallthrough
        case abiStepIntReg:
            storeRcvr(rcvr, unsafe.Pointer(&regArgs.Ints[st.ireg]))
        case abiStepFloatReg:
            storeRcvr(rcvr, unsafe.Pointer(&regArgs.Floats[st.freg]))
        default:
            panic("unknown ABI parameter kind")
        }
        inStart = 1
    }

    // Handle arguments.
    for i, v := range in {
        v.mustBeExported()                                      // ← ANOTHER runtime check per arg
        targ := toRType(t.In(i))
        v = v.assignTo("reflect.Value.Call", &targ.t, nil)      // ← type coercion + safety check

    stepsLoop:
        for _, st := range abid.call.stepsForValue(i + inStart) {
            switch st.kind {
            case abiStepStack:
                // Copy values to the "stack."
                addr := add(stackArgs, st.stkOff, "precomputed stack arg offset")
                if v.flag&flagIndir != 0 {
                    typedmemmove(&targ.t, addr, v.ptr)          // ← memcpy with GC write barrier
                } else {
                    *(*unsafe.Pointer)(addr) = v.ptr             // ← raw pointer store
                }
                break stepsLoop

            case abiStepIntReg, abiStepPointer:
                // Copy values to "integer registers."
                if v.flag&flagIndir != 0 {
                    offset := add(v.ptr, st.offset, "precomputed value offset")
                    if st.kind == abiStepPointer {
                        // Duplicate this pointer in the pointer area of the
                        // register space. Otherwise, there's the potential for
                        // this to be the last reference to v.ptr.
                        regArgs.Ptrs[st.ireg] = *(*unsafe.Pointer)(offset)
                    }
                    intToReg(&regArgs, st.ireg, st.size, offset)
                } else {
                    if st.kind == abiStepPointer {
                        regArgs.Ptrs[st.ireg] = v.ptr
                    }
                    regArgs.Ints[st.ireg] = uintptr(v.ptr)
                }

            case abiStepFloatReg:
                // Copy values to "float registers."
                if v.flag&flagIndir == 0 {
                    panic("attempted to copy pointer to FP register")
                }
                offset := add(v.ptr, st.offset, "precomputed value offset")
                floatToReg(&regArgs, st.freg, st.size, offset)

            default:
                panic("unknown ABI part kind")
            }
        }
    }
```

**For each argument**, this code:
1. Calls `mustBeExported()` — flag check
2. Calls `assignTo()` — type coercion with safety checking
3. Iterates over `abiSteps` to determine: does this arg go on the stack or in a register? Which register?
4. Copies bytes via `typedmemmove` (which includes a **GC write barrier**) or raw pointer store

Compare this to DirectCall: `a0 := args[0].String()` — one type assertion, zero copies, zero write barriers.

#### The actual `reflectcall` invocation

```go
// Source: reflect/value.go (Go 1.23) — continued

    // ── PHASE 8: Align frame and set up spill space ──
    frameSize = align(frameSize, goarch.PtrSize)
    frameSize += abid.spill

    // Mark pointers in registers for the return path.
    regArgs.ReturnIsPtr = abid.outRegPtrs

    // ── PHASE 9: THE ACTUAL CALL ──
    call(frametype, fn, stackArgs, uint32(frametype.Size()),
         uint32(abid.retOffset), uint32(frameSize), &regArgs)
```

Where `call` is an assembly trampoline:

```go
// Source: reflect/value.go
// call is implemented in assembly (asm_amd64.s, asm_arm64.s, etc.)
//
//go:noescape
func call(stackArgsType *abi.Type, f unsafe.Pointer, stackArgs unsafe.Pointer,
    stackArgsSize, stackRetOffset, frameSize uint32, regArgs *abi.RegArgs)
```

The assembly (`asm_amd64.s`) does:
1. Set up a new stack frame of `frameSize` bytes
2. Copy `stackArgs` to the new frame
3. Load `regArgs.Ints[0..N]` into registers RAX, RBX, RCX, RDI, RSI, R8-R11 (Go ABI internal)
4. Load `regArgs.Floats[0..N]` into XMM0-XMM14
5. **Indirect CALL** through `fn` — `CALL fn` (the function pointer)
6. Copy registers back into `regArgs`
7. Return

The **indirect call** is critical: the CPU's branch predictor cannot predict the target of an indirect call as well as a direct call. This typically adds 5–15 cycles of branch misprediction penalty.

#### Return value unmarshaling

```go
// Source: reflect/value.go (Go 1.23) — continued

    // ── PHASE 10: Unmarshal return values ──
    var ret []Value
    if nout == 0 {
        if stackArgs != nil {
            typedmemclr(frametype, stackArgs)
            framePool.Put(stackArgs)                // recycle frame (no-return-value case only)
        }
    } else {
        if stackArgs != nil {
            // Zero the now unused input area of args,
            // because the Values returned by this function contain pointers to the args object,
            // and will thus keep the args object alive indefinitely.
            typedmemclrpartial(frametype, stackArgs, 0, abid.retOffset)
        }

        // Wrap Values around return values in args.
        ret = make([]Value, nout)                   // ← THIRD heap allocation!
        for i := 0; i < nout; i++ {
            tv := t.Out(i)
            if tv.Size() == 0 {
                ret[i] = Zero(toRType(tv))
                continue
            }
            steps := abid.ret.stepsForValue(i)

            if st := steps[0]; st.kind == abiStepStack {
                // This value is on the stack. Just point into the allocated frame.
                fl := flagIndir | flag(tv.Kind())
                ret[i] = Value{tv, add(stackArgs, st.stkOff, "tv.Size() != 0"), fl}
                // Note: this introduces false sharing between results -
                // if any result is live, they are all live.
                continue
            }

            // Handle pointers passed in registers.
            if !tv.IfaceIndir() {
                // Pointer-valued data gets put directly into v.ptr.
                ret[i] = Value{tv, regArgs.Ptrs[steps[0].ireg], flag(tv.Kind())}
                continue
            }

            // All that's left is values passed in registers that we need to
            // create space for and copy values back into.
            //
            // TODO(mknyszek): We make a new allocation for each register-allocated
            // value, but previously we could always point into the heap-allocated
            // stack frame.
            s := unsafe_New(tv)                     // ← POSSIBLE 4th allocation per return value!
            for _, st := range steps {
                switch st.kind {
                case abiStepIntReg:
                    offset := add(s, st.offset, "precomputed value offset")
                    intFromReg(&regArgs, st.ireg, st.size, offset)
                case abiStepPointer:
                    s := add(s, st.offset, "precomputed value offset")
                    *((*unsafe.Pointer)(s)) = regArgs.Ptrs[st.ireg]
                case abiStepFloatReg:
                    offset := add(s, st.offset, "precomputed value offset")
                    floatFromReg(&regArgs, st.freg, st.size, offset)
                case abiStepStack:
                    panic("register-based return value has stack component")
                default:
                    panic("unknown ABI part kind")
                }
            }
            ret[i] = Value{tv, s, flagIndir | flag(tv.Kind())}
        }
    }

    return ret
}
```

Note the comment from the Go team themselves:
> *"TODO(mknyszek): We make a new allocation for each register-allocated value"*

For register-returned values (common with Go 1.17+ ABI), **each return value** may trigger an additional `unsafe_New` allocation! For `strconv.Atoi` which returns `(int, error)`:
- `int` → returned in register → `unsafe_New` to create storage → pointed to by `ret[0]`
- `error` → returned in register → `unsafe_New` to create storage → pointed to by `ret[1]`

That's potentially **5 allocations** per call: `[]reflect.Value` input, stack frame, `[]Value` output, + 2 return value storage.

### 3.5 — Return Value Unboxing: `reflect.Value` → `value.Value` (per result)

**DirectCall:**
```go
return value.MakeInt(int64(r0))  // Store int in num field — ~1 ns, zero alloc
```
`MakeInt` is:
```go
// Source: model/value/value.go
func MakeInt(v int64) Value {
    return Value{kind: KindInt, num: v}  // 32-byte struct, stack-allocated
}
```
No allocation. No reflection. Just fill two fields of a struct.

**reflect.Call:**
```go
// vm/call.go:410
v.push(value.MakeFromReflect(out[0]))
```

`MakeFromReflect` (model/value/value.go:349):
```go
// Source: model/value/value.go
func MakeFromReflect(rv reflect.Value) Value {
    if !rv.IsValid() {
        return MakeNil()
    }
    kind := rv.Kind()
    switch kind {
    case reflect.Int:
        return MakeInt(rv.Int())  // rv.Int() reads through the reflect.Value indirection
    // ... 20+ cases ...
    }
}
```

`rv.Int()` internally:
```go
// Source: reflect/value.go (Go 1.23)
func (v Value) Int() int64 {
    f := v.flag
    k := f.kind()
    switch k {
    case Int:
        return int64(*(*int)(v.ptr))        // ← pointer dereference through v.ptr
    case Int8:
        return int64(*(*int8)(v.ptr))
    case Int16:
        return int64(*(*int16)(v.ptr))
    case Int32:
        return int64(*(*int32)(v.ptr))
    case Int64:
        return *(*int64)(v.ptr)
    }
    panic(&ValueError{"reflect.Value.Int", v.kind()})
}
```

Every access goes: `flag` → extract kind bits → `switch` → dereference `v.ptr`. Two levels of indirection (flag check + pointer deref) vs DirectCall's zero.

**Cost**: ~5–10 ns per result due to flag checking and pointer chasing.

### 3.6 — Branch Predictor and Inlining Defeat

This is a **systemic** overhead that compounds with all the above.

**DirectCall:**
```go
func direct_strconv_Atoi(args []value.Value) value.Value {
    a0 := args[0].String()
    r0, r1 := strconv.Atoi(a0)
    return value.MakeValueSlice(...)
}
```
The Go compiler can:
- **Inline** `args[0].String()` (it's a simple type assertion)
- **Inline** `value.MakeInt()` (it's a struct literal)
- **Devirtualize** the call to `strconv.Atoi` (direct call, known at compile time)
- The CPU's **branch predictor** sees a simple linear path with no indirect jumps

**reflect.Call:**
- `reflect.Value.Call` is a **megamorphic call site** — it calls different functions every time. The CPU branch predictor must handle:
  - Indirect call through the assembly `call` trampoline → branch misprediction
  - Multiple `switch` statements inside `assignTo`, the `abiStep` loop, `intToReg` → branch mispredictions
- **Nothing is inlinable** — `reflect.Call`, `assignTo`, `typedmemmove` are all too complex for the Go inliner (Go's inliner has a budget of ~80 AST nodes; `Value.call` has 500+)
- **Pointer chasing**: `reflect.Value.typ_ → abi.Type → Kind_`, `reflect.Value.ptr → data` — each pointer chase can be an L1 cache miss (L1 miss penalty: ~4 ns on modern x86)

## 4. Side-by-Side Cost Breakdown

For a call to `strconv.Atoi("123")` (1 string arg, 2 return values):

| Step | DirectCall | reflect.Call | Ratio |
|---|---|---|---|
| Pre-checks | — (none) | `mustBe(Func)` + `mustBeExported()` → 3 ns | ∞ |
| Arg extraction | `args[0].String()` → 2 ns | `ToReflectValue()` + `ValueOf()` → 20 ns | 10x |
| Intermediate slice alloc | — (none) | `make([]reflect.Value, 1)` → 30 ns | ∞ |
| Argument type checking | — (compile time) | `AssignableTo()` per arg → 20 ns | ∞ |
| ABI layout | — (static) | `funcLayout` sync.Map lookup → 10 ns | ∞ |
| Call frame alloc | — (none) | `unsafe_New(frametype)` → 30 ns | ∞ |
| Arg marshaling | — (register pass) | `assignTo` + `typedmemmove` per arg → 15 ns | ∞ |
| The actual call | Direct `CALL` → 3 ns | Indirect via asm trampoline → 15 ns | 5x |
| Return slice alloc | — (none) | `make([]Value, 2)` → 30 ns | ∞ |
| Return value storage | — (none) | `unsafe_New` per register-returned value → 30 ns | ∞ |
| Result extraction | `MakeInt(r0)` → 2 ns | `MakeFromReflect(out[0])` → 10 ns | 5x |
| GC pressure | 0 allocs | 3–5 allocs (~200 bytes) | ∞ |
| **Total overhead** | **~10–15 ns** | **~210–280 ns** | **~18x** |

*Note: The actual function body of `strconv.Atoi` takes ~30–50 ns, so the total wall time is ~40–65 ns (DirectCall) vs ~240–330 ns (reflect.Call). As the function body gets more expensive, the ratio approaches 1x because the overhead becomes a smaller fraction.*

## 5. The Allocation Problem in Detail

Every `reflect.Call` for a function with return values creates **3–5 garbage objects**:

```
Allocation 1: []reflect.Value   for input args          24 bytes × numArgs
Allocation 2: stackArgs frame   via unsafe_New(frametype) ~40–100 bytes (NOT poolable if nout > 0!)
Allocation 3: []reflect.Value   for output results       24 bytes × numResults
Allocation 4+: unsafe_New(tv)   per register-returned value  (size of return type each)
```

The Go team's own comment in the source acknowledges this regression:
```go
// TODO(mknyszek): We make a new allocation for each register-allocated
// value, but previously we could always point into the heap-allocated
// stack frame. This is a regression that could be fixed by adding
// additional space to the allocated stack frame and storing the
// register-allocated return values into the allocated stack frame and
// referring there in the resulting Value.
```

At 1000 calls/sec, this produces:
- ~3000–5000 allocations/sec
- ~200–500 KB of garbage/sec
- Additional GC scan work (`reflect.Value` contains pointers → cannot be `noScan`)

DirectCall produces **zero** allocations for primitive types (int, string, bool, float). The `value.Value` tagged union stores primitives inline in the 32-byte struct, which lives on the stack.

## 6. Why Go's Reflection Is Fundamentally Slow

Go's `reflect` package was designed for **correctness and safety**, not performance:

1. **No specialization**: `reflect.Call` uses the same code path for `func()`, `func(int) int`, and `func(string, ...interface{}) ([]byte, error)`. There's no JIT compilation or caching of specialized call stubs. Every call goes through the full 10-phase pipeline.

2. **No register-direct path**: Even on Go 1.17+ with register-based ABI, `reflect.Call` must marshal args into a memory buffer (`regArgs` struct + `stackArgs` buffer), then have the assembly trampoline copy them into registers. A direct call puts args in registers immediately — the compiler knows the calling convention at compile time.

3. **Full type system at runtime**: Every `reflect.Value` carries a complete type descriptor (`*abi.Type` → 48+ bytes). Operations on it require pointer chasing through the type descriptor to find `Kind_`, `Size_`, `Hash`, `Equal`, etc.

4. **No escape analysis**: The Go compiler cannot prove that `[]reflect.Value` allocations don't escape — they are passed to `reflect.Value.call` which is too complex for escape analysis (500+ AST nodes). So they **always heap-allocate**. DirectCall args are `[]value.Value` which the wrapper owns entirely and may be stack-allocated.

5. **No devirtualization**: The Go compiler can devirtualize and inline direct function calls. But `reflect.Call` is an opaque call through `unsafe.Pointer` → assembly trampoline → `fn`. The compiler cannot see through it, so no inlining or optimization is possible.

6. **GC write barriers on every arg**: `typedmemmove` in the argument marshaling loop inserts GC write barriers for each argument that contains pointers. This ensures GC correctness but adds ~2–5 ns per pointer-containing argument. DirectCall has no write barriers — it reads args directly from the `value.Value` slice and passes them as Go values.

## 7. Gig's Architecture Decision

Gig uses **code generation** (`cmd/gig/gentool/`) to produce typed DirectCall wrappers at build time for every exported function and method in registered packages. This is the same strategy used by:

- **gRPC-Go**: generates typed client/server stubs instead of using reflection
- **encoding/json** (Go 1.21+): moving toward generated encoders
- **Wire** (DI): generates typed provider functions

The trade-off is **code size** (each wrapper is 5–15 lines of generated Go) for **10–18x runtime performance** on every external call. Since Gig is an interpreter that may call external functions millions of times in a loop, this trade-off is overwhelmingly worthwhile.

### When reflect.Call is still used

DirectCall wrappers are not generated for:
- Methods whose parameter types can't be expressed in the wrapper (e.g., unexported types)
- Dynamic dispatch through interfaces where the concrete method isn't known at compile time
- Closures passed as arguments that need `reflect.MakeFunc` to bridge interpreted ↔ native

In these cases, Gig falls back to `callExternalReflect` or `callExternalMethodReflect` — which use the full reflection path described above.
