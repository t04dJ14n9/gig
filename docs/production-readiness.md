# Gig Production Readiness & External Library Compatibility

## Architecture Overview

Gig is a Go interpreter that executes bytecode compiled from Go source via SSA.
It registers external Go standard library and third-party packages so interpreted
code can call real Go functions.

### Key Subsystems

| Package | Role |
|---------|------|
| `bytecode/` | Pure structs (no interfaces) |
| `vm/` | `StatefulExecutor` interface, VM pool, goroutine tracker |
| `compiler/` | SSA-based Go source to bytecode compiler |
| `importer/` | `PackageRegistry`, `PackageLookup` interfaces, type conversion |
| `value/` | `Value` type system, `ClosureExecutor`, `MethodResolver` |
| `stdlib/` | Declares available stdlib packages via `pkgs.go` imports |
| `stdlib/packages/` | Generated registration code (auto-generated by `gig gen`) |
| `runner/` | Execution orchestration (VM pool, global state, timeout) |

## fmt Wrapper Architecture

The `fmt` package requires special handling because interpreter-synthesized structs
are backed by `reflect.StructOf` types that do not carry Go named-type identity
or methods. Without intervention, `fmt.Sprintf("%T", myStruct)` would report an
anonymous struct type and `fmt.Stringer`/`fmt.GoStringer` methods defined in the
script would be invisible to Go's `fmt` package.

### How It Works

1. **`value.FmtWrap()`** - Generated `fmt.*` DirectCall wrappers pass interface
   and variadic args through this function. It detects interpreter structs by
   their registered `reflect.Type` name or `gig` struct tag and wraps them in
   `gigStructWrapper`.

2. **`gigStructWrapper`** - Implements `fmt.Formatter`, `fmt.Stringer`, and
   `fmt.GoStringer`:
   - `%v` / `%s`: Prints struct fields, or calls `String()` if the interpreted
     type implements it
   - `%#v`: Detailed format with field names
   - `%T`: Reports the interpreter type name from the wrapper metadata

3. **`value.SprintfExtern()`** - Standard `fmt.Sprintf` bypasses
   `fmt.Formatter` for `%T`. This function manually parses the format string and
   substitutes the correct type name for `gigStructWrapper` args.

4. **Method-aware formatting** - `FmtWrap` resolves `String()`, `Error()`, and
   `GoString()` lazily through the VM method resolver only when `fmt` asks for
   the corresponding interface.

### Known Limitations

- **Reflect-based structs** (created via `reflect.StructOf` by the interpreter)
  must always be wrapped before passing to `fmt`.
- **`%T` format verb** requires the `value.SprintfExtern` slow path.
- **Interface method dispatch** works only through the registered wrapper/proxy
  paths. Plain third-party interface parameters remain rejected by default.

## Safe External Libraries

These libraries work correctly because they only perform pure computation.

### String & Bytes
`strings`, `bytes`, `strconv`, `unicode`, `unicode/utf8`, `unicode/utf16`, `bufio`

### Formatting
`fmt` (special: requires `gigStructFormatter` wrapper for interpreter structs)

### Math
`math`, `math/big`, `math/bits`, `math/cmplx`, `math/rand/v2`

### Collections & Sorting
`sort`, `cmp`, `slices`, `maps`, `container/heap`, `container/list`, `container/ring`

### Encoding & Serialization
`encoding/json`, `encoding/base64`, `encoding/hex`, `encoding/csv`,
`encoding/asn1`, `encoding/binary`, `encoding/ascii85`, `encoding/pem`

> **Note:** `encoding/xml` and `encoding/gob` are intentionally not supported.
> XML requires named struct types (incompatible with `reflect.StructOf`).
> GOB has similar limitations with recursive types. Use `encoding/json` instead.

### Regular Expressions
`regexp`, `regexp/syntax`

### Hashing (non-crypto)
`hash/adler32`, `hash/crc32`, `hash/crc64`, `hash/fnv`, `hash/maphash`

### Cryptographic Hashing (pure computation)
`crypto/hmac`, `crypto/md5`, `crypto/sha1`, `crypto/sha256`, `crypto/sha512`,
`crypto/subtle`, `crypto/aes`, `crypto/cipher`, `crypto/des`, `crypto/elliptic`,
`crypto/rc4`

### Time
`time` (full support, `time.Duration` DirectCall works)

### Concurrency
`context`, `sync`

### HTML & Text
`html`, `text/scanner`, `text/tabwriter`

### URL & IP (pure string ops)
`net/url`, `net/netip`

### Compression & Archives (byte streams)
`compress/gzip`, `compress/zlib`, `compress/flate`, `compress/bzip2`, `compress/lzw`,
`archive/tar`, `archive/zip`

### MIME
`mime`, `mime/multipart`, `mime/quotedprintable`

### Image (partial)
`image/color` (pure math)

### Path (pure string ops)
`path`

### Error Handling
`errors` (`errors.New`, `errors.Is`, `errors.As`, `fmt.Errorf` with `%w`)

### Index
`index/suffixarray`

## Unsafe External Libraries (Disabled by Default)

These are disabled in `stdlib/pkgs.go` to maintain sandbox isolation.

### Filesystem
`os`, `os/exec`, `os/signal`, `os/user`, `path/filepath`, `io/fs`

### Network
`net`, `net/http`, `net/mail`, `net/smtp`, `net/rpc`

### Logging
`log`, `log/slog` (writes to `os.Stderr`)

### Templates
`html/template`, `text/template` (can call arbitrary methods), `embed`

### Runtime
`runtime`, `syscall`

### Asymmetric Crypto
`crypto/rand` (reads `/dev/urandom`), `crypto/ecdsa`, `crypto/ed25519`,
`crypto/rsa`, `crypto/tls`, `crypto/x509`

### Database
`database/sql`

## Production Recommendations

1. **Keep `stdlib/pkgs.go` minimal** - Only import packages that interpreted code
   actually needs. Each import increases startup time and memory.

2. **Test thoroughly** - Run `go test ./...` in the root module and in nested
   modules such as `cmd/gig`, `tests/thirdparty_external`, `examples/*`, and
   `benchmarks` before deploying.

3. **Keep third-party boundaries explicit** - Script-defined types are rejected
   at third-party calls unless they are passed through a registered interface
   proxy or the host deliberately opts into `WithAllowUnsafeTypePass()`.

4. **Method dispatch** - The `MethodResolverFunc` registry uses `sync.Map` with
   program pointer keys. Clean up programs properly to avoid memory leaks.

5. **Concurrency** - The VM pool is goroutine-safe. Each `Run` call gets its own
   VM instance. Use `WithStatefulGlobals()` for persistent globals.

6. **Timeout** - Always use `RunWithContext` with a timeout to prevent infinite
   loops from hanging the interpreter.
